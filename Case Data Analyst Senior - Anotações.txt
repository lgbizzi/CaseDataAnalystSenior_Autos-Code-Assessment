Case Data Analyst Senior:
Link Github: https://github.com/lgbizzi/CaseDataAnalystSenior_Autos-Code-Assessment
- Com base na documentação que foi informada "Autos Code Assessment";
- Com o objetivo de desenvolver um sistema de IA que analisa os dados de vendas, relacionados a:
	-> Veículos;
	-> Peças;
	-> Serviços;
	-> Inventório;
- Tracei um plano de ação para auxiliar nas abordagens:
	1 - Identificar padrões de vendas e tendências de serviços;
	2 - Prever necessidades de estoque e reduzir ineficiências operacionais;
	3 - Gerar insights para apoiar na tomada de decisão para a gestão;
	4 - Entregar relatórios automatizados, sumarizando insights em linguagem natural, através de APIs OpenAI;


Em anexo está o plano de ação tratado, pensando em uma implementação em equipe, ao longo de algumas semanas de Projeto.
		-> Pensando na Arquitetura medalhão, com as camadas:
			* BRONZE (Armazenamento de dados brutos);
			* SILVER (Armazenamento de dados limpos, tratados e organizados);
			* GOLD (Armazenamento de dados relacionados e enriquecidos);

Como não seria possível implementar todos os pontos previstos, dentro do prazo de resolução do Case, para o desenvolvimento do MVP, foquei nos pontos:
	1 - Entendimento do Dicionário de Dados (Data Overview) apresentado, em relação a estrutura dos arquivos .csv;
	2 - Através de Docker Container, montar um Banco de Dados (BD) Oracle para armazenamento dos dados;
	3 - Com base no Dicionário de Dados (Data Overview), criar as tabelas no Banco de Dados Oracle;
	4 - Desenvolvimento de automações para ETL dos dados, em Python;
		-> Devido ao tempo para resolução do case;
		-> Apliquei alguns tratamentos de inconsistências de dados nessa etapa:
			* Acentuação quebrada;
			* Campos de "Cod_Concessionaria" e "Cod_Filial" ausentes em algumas bases;
			* "Cod_Filial" fora do padrão (estava com tipo data);
		-> Foram feitos antes de subir ao Banco de Dados;

	5 - Desenvolvimento de Agente de IA (LLM - OpenAI) para análise dos dados armazenados:
		-> Desenvolvimento em Python;
		-> Uso de bibliotecas Langchain;
		-> Através de API OpenAI;
		-> Conectado ao BD Oracle criado;

	6 - Desenvolvimento de visualizações interativas:
		-> Desenvolvimento em Python;
		-> Uso de Streamlit;
		-> Conectado ao BD Oracle criado;


1 - Entendimento do Dicionário de Dados (Data Overview) apresentado, em relação a estrutura dos arquivos .csv;
#### 1.1 Estoque de Veículos
- **Valores nulos:** 10 registros em colunas críticas
- **Duplicatas:** 3 registros duplicados
- **Estrutura:** Colunas repetidas (Data_de_Entrada aparece 3 vezes)
- **Ação:** Limpeza, remoção de duplicatas, deduplicate colunas

#### 1.2 Histórico de Vendas de Veículos
- **Valores nulos:** 41.772 nulos (coluna "Unnamed: 27" - vazia)
- **Duplicatas:** 1.417 registros duplicados
- **Margem anômala:** 1,14% muito baixa (investigar cálculos)
- **Ação:** Remover coluna vazia, deduplicar, validar fórmulas

#### 1.3 Histórico de Serviços
- **Lucro > Receita:** 114% de margem CRÍTICO
- **Possível causa:** Lucro acumulado ou estrutura diferente
- **Ação:** Investigação urgente da semântica dos dados

#### 1.4 Estoque de Peças
- **Status:** Limpo (sem problemas aparentes)
- **Validação necessária:** Datas, campos de obsolescência
- **Ação:** Confirmar campos críticos, validar ranges

#### 1.5 Histórico de Vendas de Peças
- **Margens anômalas:** Peças com lucro negativo ou margem >100%
- **Zeros:** Custo da peça zerado em registros (verificar se correto)
- **Negativas:** Transações negativas (devoluções/cancelamentos não claramente marcadas)
- **Incoerências:** Mesmo item com custos diferentes por filial/período
- **Vendedor/Comprador:** Alguns registros com mesmo nome (venda interna?)
- **Ação:** Padronizar margens por canal, investigar negativas, validar custo



2 - Através de Docker Container, montar um Banco de Dados (BD) Oracle para armazenamento dos dados:
#### 2.1 Obter Container Oracle
docker run --name oracle_db_container -p 1521:1521 -e ORACLE_PWD=123@Troca -v oracle_db_data:/opt/oracle/oradata container-registry.oracle.com/database/free:latest

#### 2.2 Após instalação, executar Container Docker e iniciar serviço SQLPlus
docker exec -it --user oracle oracle_db_container bash
sqlplus sys/"123@Troca"//localhost:1521/FREE as sysdba

#### 2.3 Listar PDBs disponíveis
SHOW PDBS;

#### 2.4 Conectar no PDB correto (FREEPDB1)
ALTER SESSION SET CONTAINER = FREEPDB1;

#### 2.5 Confirmar que está no PDB certo
SHOW CON_NAME;

#### 2.6 Criar usuário LOCAL no PDB
CREATE USER autos_code_dba IDENTIFIED BY "123@Troca" DEFAULT TABLESPACE USERS QUOTA UNLIMITED ON USERS;

#### 2.7 Grants DBA completos no PDB
GRANT DBA TO autos_code_dba;

#### 2.8 Verificar criação
SELECT username, account_status FROM dba_users WHERE username = 'AUTOS_CODE_DBA';

-- BRZ_ESTOQUE_PECAS
CREATE TABLE BRZ_ESTOQUE_PECAS (
    ID_ESTOQUE_PECA          NUMBER GENERATED BY DEFAULT AS IDENTITY,
    COD_CONCESSIONARIA       VARCHAR2(10)   NOT NULL,
    COD_FILIAL               VARCHAR2(10)   NOT NULL,
    NOME_CONCESSIONARIA      VARCHAR2(100),
    NOME_FILIAL              VARCHAR2(100),
    MARCA_FILIAL             VARCHAR2(50),
    VALOR_PECA_ESTOQUE       NUMBER(18,2),
    QTDE_PECA_ESTOQUE        NUMBER(18,2),
    DESCRICAO_PECA           VARCHAR2(200),
    CATEGORIA_PECA           VARCHAR2(100),
    DT_ULTIMA_VENDA_PECA     DATE,
    DT_ULTIMA_ENTRADA_PECA   DATE,
    PECA_OBSOLETA_FLAG       VARCHAR2(3),
    TEMPO_OBSOLETA_DIAS      NUMBER(10),
    MARCA_PECA               VARCHAR2(100),
    CODIGO_PECA_ESTOQUE      VARCHAR2(50),
    CONSTRAINT PK_BRZ_ESTOQUE_PECAS
        PRIMARY KEY (ID_ESTOQUE_PECA)
);

-- Índices recomendados
CREATE INDEX IX_BRZ_EST_PECAS_FILIAL
